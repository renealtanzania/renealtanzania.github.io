Overview

This documentation provides detailed information about Reneal server build-up procedures, VPN creation processes, command lines and related features. This should only be done by someone very familiar with using the Linux command line. Instructions must be followed exactly.

Before rebuilding a server, it's critical to prepare VPN configuration files in advance including network settings and firewall rules. By creating the VPN configuration files beforehand, you ensure a smooth transition and can immediately restore network connectivity after the rebuild is complete.

Some System Features

  • Robust set of Applications: such as (office suite, photo/video editing, webpage design, desktop publishing, CAD, programming, etc.).
  • Educational programs: Including (math, science, English, geography).
  • Extensive educational content: Consisting (books, movies, offline Wikipedia, past National Exam papers, Tanzania curriculum notes, science simulations, etc.).
  • Epoptes: For classroom management program so teacher can monitor student work or share. their screen to students (very convenient if school does not have a projector), shared video streaming capability.
  • Permission-controlled content sharing area: For TeachersShared and AllUsersShared file storage.

Getting Started

To begin creating VPN files, follow these steps:

  1. You must create the VPN files in the MASTER Account on a Reneal server.
  2. You will then install the created files onto each newly built server.
  3. All necessary files required in build-up process will be on the VPN flash drive.

Authentication

All API requests require authentication. We support both API keys and OAuth 2.0 for secure access.

// Example API key authentication curl -H "Authorization: Bearer YOUR_API_KEY" \ https://api.example.com/v1/users

API Endpoints

Our RESTful API provides access to all core platform functionality through standardized endpoints.

Endpoint Method Description Parameters
/users GET Retrieve list of users limit offset
/users/{id} GET Retrieve specific user id
/users POST Create new user name email role
/projects GET Retrieve list of projects status owner

Code Examples

Here are some common implementation examples to help you get started quickly.

JavaScript Example

// Fetch user data async function getUser(userId) { const response = await fetch(`https://api.example.com/v1/users/${userId}`, { headers: { 'Authorization': `Bearer ${API_KEY}`, 'Content-Type': 'application/json' } }); if (!response.ok) { throw new Error(`HTTP error! status: ${response.status}`); } return await response.json(); } // Create new project async function createProject(projectData) { const response = await fetch('https://api.example.com/v1/projects', { method: 'POST', headers: { 'Authorization': `Bearer ${API_KEY}`, 'Content-Type': 'application/json' }, body: JSON.stringify(projectData) }); return await response.json(); }

Python Example

import requests # Configure API endpoint and headers BASE_URL = "https://api.example.com/v1" HEADERS = { "Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json" } # Get user information def get_user(user_id): response = requests.get( f"{BASE_URL}/users/{user_id}", headers=HEADERS ) response.raise_for_status() return response.json() # Create new task def create_task(task_data): response = requests.post( f"{BASE_URL}/tasks", json=task_data, headers=HEADERS ) response.raise_for_status() return response.json()
