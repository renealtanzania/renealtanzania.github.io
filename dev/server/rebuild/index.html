Introduction

Welcome to the Official Reneal Documentation for Reneal Server Build-Up. This documentation provides detailed information about Reneal server build-up procedures, VPN creation processes, command lines and related features.

Important Notice

This should only be done by someone very familiar with using the Linux command line. Instructions must be followed exactly. Before rebuilding a server, it's critical to prepare VPN configuration files in advance including network settings and firewall rules.

By creating the VPN configuration files beforehand, you ensure a smooth transition and can immediately restore network connectivity after the rebuild is complete.

System Features

  • Robust set of Applications: Office suite, photo/video editing, webpage design, desktop publishing, CAD, programming, and more
  • Educational programs: Math, science, English, geography
  • Extensive educational content: Books, movies, offline Wikipedia, past National Exam papers, Tanzania curriculum notes, science simulations
  • Epoptes: Classroom management program allowing teachers to monitor student work or share their screen to students (very convenient if school does not have a projector), shared video streaming capability
  • Permission-controlled content sharing: TeachersShared and AllUsersShared file storage areas

Getting Started

To begin creating VPN files for WireGuard, follow these steps:

  1. 1 You must create the VPN files in the MASTER Account on a Reneal server
  2. 2 You will then install the created files onto each newly built server
  3. 3 All necessary files required in build-up process will be put in the VPN flash drive

VPN Creation Procedures

Setup Environment

Boot up the server that you will be using to create the VPN files (you can use the JR Reneal server). Sign in to the master account and insert the VPN USB flash drive.

Note: You should only have to copy the VPN creation files one time, the first time you do this process on the JR Reneal server.

Copy the VPN creation files from the VPN USB drive to the JR Reneal server:

Shell
cd ~/CodeDevelopment/newSystemBuild
sudo cp -pR /media/master/RENEALVPN/makeWireguardVpnFiles .
cd makeWireguardVpnFiles
chmod 744 buildWgConf0.sh
chmod 744 buildWgConf1.sh

Create School List

Create a text file in the ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles directory. An example file name might be 2025Aschools_wg.(txt)

In this text file, put one school per row, with new school names added to "-main-server". Make sure there is no space in the entry, and make sure there is no extra line at the end of the text file.

Example format:

Text File
amsha-main-server
mukulat-main-server
sanya_juu-main-server

The file path should be: ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles

Edit Configuration Files

Edit the bash files that build the wg0.conf and wg1.conf files for each school. You need to edit the starting address for wireguard.

Critical: Coordinate with Rene

You must hand-edit the starting address (ADDR) for WireGuard in the two script files: buildWgConf0.sh and buildWgConf1.sh. You must discuss with Rene to get the correct starting address each time!

Example address assignments:

  • For 2024B, the wireguard starting address was 117 (Eleven servers)
  • For 2025A, the wireguard starting address was 128 (Ten servers)
  • For 2025B, the wireguard addresses will start with 139

Build VPN Files

Now you will run the script to build the new wg0.conf files for each school:

Build wg0 configuration files:

Go to the wg0 folder and run the build script:

Shell
cd wg0 # Confirm you are in the correct directory # ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/wg0
cat ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/2025Aschools_wg | \ ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/buildWgConf0.sh

This will create the files and put them in: ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/wg0

Build wg1 configuration files:

Repeat the process for wg1:

Shell
cd ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/wg1
cat ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/2025Aschools_wg | \ ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/buildWgConf1.sh

This will create the files and put them in: ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/wg1

Copy Files to Flash Drive

Copy the VPN files you have just created to the VPN flash drive so they can be installed on each individual new server:

Shell
cp -pRu ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/wg0 .
cp -pRu ~/CodeDevelopment/newSystemBuild/makeWireguardVpnFiles/wg1 .

Edit the program install_info_wireguard.py with the school name information:

Important: You must hand edit install_info_wireguard.py with the names of the schools.

Install Files on Each Server

NOTE: This Step Comes when you are Rebuilding the Server after you have finished Setting a Unique Configuration just before Completing Final Configuaration as highlighted on the Server Build-Up Process below.

For each new server you are building, you will install the wireguard files:

  1. 1 Boot the new server
  2. 2 Sign in to the master account
  3. 3 Insert the VPN USB drive
  4. 4 Run the installation program
Shell
sudo python3 install_info_wireguard.py

You will be prompted to enter the name of the school for this new server when you run the program, which will be matched against the list that you hand-edited in install_info_wireguard.py

This program will put the VPN files in /etc/wireguard on your new server.

Add the newly-created VPN addresses to the full list of hosts.

You will need to inform Rene of the school names and the VPN wireguard addresses (10.9.0.##). She will add them to the full list of hosts in the “hosts” file.

Server Build-Up Procedures

Server build-up process comes next after you have the VPN files ready. This section covers the complete 6-step process for building a new Reneal server.

1 Prepare VPN Files

Before starting the server build, ensure you have completed the VPN file creation process as described in the VPN Creation Procedures section above.

2 Set BIOS Configuration

Access the BIOS on the new server:

  • Press DEL to enter BIOS for most desktop servers
  • Press F2 for Dell MicroDesktops

BIOS Settings to Configure:

  • Use XMPPROFILE1 for the memory profile
  • Set boot device order: UEFI USB / UEFI HD
  • Set Date/Time to East Africa Time
  • Turn off wireless (for MicroDesktops only)
  • Disable Secure Boot

3 Build Server from USB

Server Image Information

The server is built from 6 .fsa files (one each for MainOS, ClientHome, ClientHomeStudents, UtilityOS, EFIPrimary, and EFISecondary). The .fsa server image files for each partition are stored in the /ServerImages directory in the Reneal Server Build USB disk.

The makeServer.py program will choose the most recent images. For the 2025A builds, the .fsa image files for ClientHome and the MainOS were dated in May of 2025.

Boot from USB:

  1. 1. Plug in your Reneal server build USB disk (make sure to use a USB3 port)
  2. 2. Boot the server:
    • Press F11 for most desktop servers
    • Press F12 for Dell MicroDesktops
  3. 3. Choose the USB disk for booting [CT1000]
  4. 4. Sign in to the master account (password is written on the USB drive)

Run the Build Process:

Open a terminal window and navigate to the build directory:

Shell
cd ~/CodeDevelopment/newSystemBuild
sudo python3 makeServer.py

Before Starting:

Read all of the text that is shown, and confirm that two disks have been identified and that the .fsa files reported are the correct ones. The program should choose the NVMe device as the primary and the hard disk as the secondary.

Build Time: The process will take approximately 2 hours. It is writing all partitions for both disks. When finished, it will report that everything is complete and you will return to the command line.

After completion: Shutdown the system, remove the Reneal Server Build USB disk, and store it somewhere safe.

4 Set Unique Configuration

A. Set Boot Order:

  1. 1. Boot the new server and enter BIOS again (press DEL)
  2. 2. Set the boot order for the two storage devices:
    • NVMe first
    • Hard disk second
  3. 3. Boot to the main OS partition

B. Configure School Information:

  1. 1. Sign in as sysadmin
  2. 2. Go to the Dashboard and type in the school name in "School Information" (for example, "Einoti Secondary School")
  3. 3. Confirm the correct number of clients (default is 20)
  4. 4. Log out of sysadmin and sign in as master

C. Configure Network Interfaces:

You will need a network switch and ethernet cable for this step.

Shell - Standard Servers
cd ~/CodeDevelopment/newSystemBuild
sudo python3 configureNetworkInterfaces.py
For Dell Micro Desktops:

You must run configureMicroNetworkInterfaces_5.py instead. This program is saved on the VPN flash drive. Copy it to ~/CodeDevelopment/newSystemBuild.

Before running, copy the files from the flashdrive to:

  • /etc/network (2 files)
  • /etc/shorewall (4 files)
Shell - Micro Desktops
cd ~/CodeDevelopment/newSystemBuild
sudo python3 configureMicroNetworkInterfaces_5.py

Network Interface Configuration:

You will be prompted to plug in and remove LAN connections, so the system can identify and label the network interfaces.

Standard Servers (4 interfaces):
  • • Motherboard LAN: Internet
  • • Dual NIC: Network switch
  • • Single NIC: Wireless router (Reneal LabWifi)
MicroDesktop (2 interfaces):
  • • Motherboard LAN: Network switch
  • • USB-LAN dongle: Wireless router (Reneal LabWifi)

Note: If there are only 3 network interfaces, it will not configure an interface for LabWifi. However, you can replace one of the single NIC cards with a dual NIC card.

Files Modified:

The configuration programs modify the following symbolic links and files:

  • /etc/network/interfaces
  • /etc/shorewall/interfaces
  • /etc/shorewall/masq
  • /etc/udev/rules.d/70-persistent-net.rules

D. Install VPN Files:

Follow the instructions in VPN Installation section to complete the VPN setup.

5 Complete Final Configuration

Run the final configuration script to complete the system build:

Shell
cd ~/CodeDevelopment/newSystemBuild
sudo ./finishNewSystemBuild.sh

This program performs:

  • Builds 5 numbered teacher accounts
  • Copies the main OS partition to each of the other 3 OS backup partitions
  • Does a full system backup

When complete, it will return to the command line. Shut down the system.

6 Testing & Verification

Setup:

  1. 1. Plug in all of the network interfaces
  2. 2. Boot the system to the main OS
  3. 3. Sign in as sysadmin

Hardware Information:

Use the Dashboard to get the Hardware Information. Take a picture and send to Rene.

Testing Checklist:

  1. 1. Run systemCheck to make sure no problems are reported.
  2. 2. Check that a client computer boots properly (student account and teacher account).
  3. 3. Make sure Kolibri is running properly.
  4. 4. Check that the Reneal Labwifi is working.
  5. 5. Check that the internet is working.
  6. 6. Reboot to each partition (4 OS partitions plus emergency partition) to make sure they all boot properly.
